# Contributing to Vernal

The project just started. The codebase exists, but there's not enough of it yet to make code contributions useful to either of us. That changes as development progresses. In the meantime, there are three ways to help that are genuinely valuable right now.

## Product Direction

Open an [Issue](https://github.com/vernal-garden/vernal/issues) if something in the spec seems wrong, missing, or over-engineered. Early decisions are cheap to change. A well-reasoned issue that catches a bad call now is worth more than ten pull requests later. The product direction is actively being shaped, and this kind of feedback is genuinely wanted, not a formality.

## Plant and Seed Data

The seed database (Cambium) is central to what makes Vernal useful, and it needs breadth. Variety data, growing notes, regional timing: this kind of contribution compounds over time and is valuable well before there's code to run. The full data contribution workflow isn't in place yet, so the current channel is [Issues](https://github.com/vernal-garden/vernal/issues). A more structured process is coming.

## SVG Plant Illustrations

The growth visualization system needs illustrated stages for a long plant list, roughly 5 to 6 stages per plant, from seed to harvest-ready. The style guide is at [vernal.garden/illustrations](https://vernal.garden/illustrations). If you're interested in contributing, open an [Issue](https://github.com/vernal-garden/vernal/issues) to say so.

## Code Contributions

Not open yet. The codebase is too early to contribute to in any useful way. When that changes, this file gets updated and a note in the repo will say what's actually open.

## A Note on Conduct

This repository is for serious, good-faith engagement. That's the expectation.
